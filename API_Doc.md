## 1. Activity Class
Controls task-level behaviors, such as locking the device into a specific application.

### `startLockTaskMode`
- **Summary:** Forces the device into a full LockTask Mode (Kiosk Mode).
- **Behavioral Detail:** 
    - **System Impact:** This API utilizes the system-level `startSystemLockTaskMode` internal call. Unlike the standard Android SDK API, it **does not require the application to be a Device Owner (DO)**.
    - **Security:** Once invoked, the system enters a permanent **LockTask Mode**. System navigation (Home, Recents) is completely disabled, preventing the user from exiting the application.
    - **Internal Mechanisms:** The `DeviceAPI library` service executes this command with system-level privileges, bypassing the standard administrative checks required for non-system applications.
    - **Version Guardrails:** Explicitly supported from library `v0.13`. Calling this on lower versions will trigger an `UnsupportedOperationException`.
- **Library Level:** `v0.13`
- **Permissions:** Requires system-level privileges granted to the `DeviceAPI Service`.
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | The context of the calling activity or service. Must be `@NonNull`. |
| `taskId` | `int` | `[Mandatory]` | The unique ID of the task to be locked. |

- **Exceptions:**

| Exception | Condition |
| :--- | :--- |
| `UnsupportedOperationException` | If the library version is below `v0.13` or hardware is incompatible. |
| `RuntimeException` | If the system service is unavailable or the IPC call fails. |

- **Example:**
```java
// Java
Activity.startLockTaskMode(context, getTaskId());
```

### `stopLockTaskMode`
- **Summary:** Releases the device from lock task mode.
- **Behavioral Detail:** Restores normal navigation (Home/Recents buttons) and allows the user to switch between applications.
- **Library Level:** `v0.13`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | The context of the calling activity or service. |

- **Example:**
```java
Activity.stopLockTaskMode(context);
```

## 2. Application Class
Utilities for application state and launcher management.

### `clearCache`
- **Summary:** Removes temporary cache files for a specific package.
- **Behavioral Detail:** 
    - **System Impact:** Deletes temporary files generated by the application to free up internal storage. Unlike `clearData`, this operation typically **does not terminate** the application's process.
    - **Scope:** This does not affect user databases, shared preferences, or any persistent settings.
- **Library Level:** `v0.4`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `packageName` | `String` | `[Mandatory]` | The target package name. |

- **Exceptions:**

| Exception | Condition |
| :--- | :--- |
| `RuntimeException` | If the system service fails to clear the cache. |

- **Example:**
```java
// Java
Application.clearCache(context, "com.example.app");
```

### `clearData`
- **Summary:** Resets an application to its initial state.
- **Behavioral Detail:** 
    - **System Impact:** Deletes all data associated with the package (databases, files, shared preferences, and settings), effectively returning the application to its clean, initial state.
    - **Process Lifecycle:** Calling this method **immediately terminates** all active processes of the target application. If the application is currently running, **the application is automatically restarted** by the system after the data wipe is complete.
- **Library Level:** `v0.4`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `packageName` | `String` | `[Mandatory]` | The target package name. |

- **Exceptions:**

| Exception | Condition |
| :--- | :--- |
| `RuntimeException` | If the operation fails within the system process. |

- **Example:**
```java
// Java
Application.clearData(context, "com.example.app");
```

### `forceStop`
- **Summary:** Immediately terminates all processes associated with a package.
- **Behavioral Detail:** 
    - **System Impact:** Forces the Android system to kill every active process and background service belonging to the specified package.
    - **Usage:** Typically used to reset an application that is in an unresponsive state or to ensure a clean start after configuration changes.
- **Library Level:** `v0.4`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `packageName` | `String` | `[Mandatory]` | The target package name. |

- **Exceptions:**

| Exception | Condition |
| :--- | :--- |
| `RuntimeException` | If the system service fails to terminate the package. |

- **Example:**
```java
// Java
Application.forceStop(context, "com.example.app");
```

### `setHomeActivity`
- **Summary:** Sets a specific activity as the default home (launcher) application.
- **Behavioral Detail:** 
    - **System Impact:** Changes the system's preferred activity for the `android.intent.category.HOME` intent.
    - **Operational Intent:** This API is provided for scenarios where the **CMS (Content Management System)** operational environment includes a home launcher feature, allowing developers to set their application as the default home launcher.
- **Library Level:** `v0.4`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `component` | `ComponentName` | `[Mandatory]` | The component of the target activity. |

- **Example:**
```java
ComponentName launcher = new ComponentName("com.solum.kiosk", "com.solum.kiosk.MainActivity");
Application.setHomeActivity(context, launcher);
```

## 3. AppOps Class
Fine-grained control over application permissions (AppOps).

### `setMode`
- **Summary:** Sets the operation mode for a specific application package.
- **Behavioral Detail:** 
    - **System Impact:** Overrides standard Android permission checks for a specific package. This is used to programmatically allow or deny sensitive operations (e.g., `"android:get_usage_stats"`) without user interaction.
- **Library Level:** `v0.1`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `appOp` | `String` | `[Mandatory]` | The AppOp string (e.g., `AppOpsManager.OPSTR_GET_USAGE_STATS`). |
| `packageName` | `String` | `[Mandatory]` | The target application package name. |
| `mode` | `int` | `[Mandatory]` | The mode to set: `0` (Allow), `1` (Ignore), `2` (Errored), `3` (Default). |

- **Example:**
```java
// Java
AppOps.setMode(context, "android:get_usage_stats", "com.example.app", 0);
```

### `setUidMode`
- **Summary:** Sets the operation mode for a specific user ID (UID).
- **Behavioral Detail:** 
    - **System Impact:** Similar to `setMode`, but targets a specific UID instead of a package name. This is useful for controlling permissions for shared UIDs or specific system components.
- **Library Level:** `v0.1`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `appOp` | `String` | `[Mandatory]` | The AppOp string. |
| `uid` | `int` | `[Mandatory]` | The target system UID. |
| `mode` | `int` | `[Mandatory]` | The mode to set: `0` (Allow), `1` (Ignore), `2` (Errored), `3` (Default). |

- **Example:**
```java
// Java
int uid = context.getPackageManager().getPackageUid("com.example.app", 0);
AppOps.setUidMode(context, "android:get_usage_stats", uid, 0);
```

### `resetAllModes`
- **Summary:** Resets all AppOps modes for a certain package.
- **Behavioral Detail:** Restores all fine-grained permissions to their system defaults for the specified user and package.
- **Library Level:** `v0.2`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `packageName` | `String` | `[Mandatory]` | Target package name. |
| `userId` | `int` | `[Optional]` | Target user ID. (Default: -1 / USER_ALL) |

- **Example:**
```java
AppOps.resetAllModes(context, "com.solum.display.sample");
```

## 4. Device Class
Provides access to hardware information and system-level status toggles.

### `getSerialNumber`
- **Summary:** Retrieves the unique hardware serial number of the device.
- **Behavioral Detail:** 
    - **Internal Mechanisms:** Accesses protected system properties or hardware registers to retrieve the factory-fused unique identifier.
- **Library Level:** `v0.5`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |

- **Example:**
```java
String serial = Device.getSerialNumber(context);
```

### `getModelName`
- **Summary:** Retrieves the hardware model name of the device.
- **Behavioral Detail:** 
    - **Internal Mechanisms:** Consults system configuration properties to return the human-readable model designation defined by the manufacturer.
- **Library Level:** `v0.10`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |

- **Example:**
```java
String model = Device.getModelName(context);
```

### `getVersion`
- **Summary:** Retrieves the SOLUM-specific system software version.
- **Behavioral Detail:** 
    - **Internal Mechanisms:** Returns the internal version string of the SOLUM system-level software components. This version follows its own lifecycle and may differ from the standard Android OS version.
- **Library Level:** `v0.10`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |

- **Example:**
```java
String version = Device.getVersion(context);
```

### `requestBugReport`
- **Summary:** Captures a comprehensive system-level bug report.
- **Behavioral Detail:** 
    - **Content:** Aggregates various system logs (logcat), kernel information, and hardware status into a single diagnostic file.
    - **Resource Lifecycle:** This is a long-running asynchronous operation. If the target file exists, it will be overwritten. If an I/O error occurs, the library attempts to delete the incomplete file.
    - **Internal Mechanisms:** Streams data from the system service via a `ParcelFileDescriptor` to the specified file using a background thread pool.
- **Library Level:** `v0.14`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `file` | `File` | `[Mandatory]` | The destination file for the bug report. |
| `executor` | `Executor` | `[Optional]` | Executor for background writing (Defaults to `AsyncTask.THREAD_POOL_EXECUTOR`). |

- **Exceptions:**

| Exception | Condition |
| :--- | :--- |
| `RuntimeException` | Triggered if IPC communication fails or an I/O error occurs while writing to the file. |

- **Example:**
```java
// Java
File targetFile = new File(context.getCacheDir(), "bugreport.txt");
ExecutorService executor = Executors.newSingleThreadExecutor();

Device.requestBugReport(context, targetFile, executor)
    .thenAccept(file -> {
        Log.i("library", "Bug report captured: " + file.getAbsolutePath());
    })
    .exceptionally(e -> {
        Log.e("library", "Failed to capture bug report", e);
        return null;
    });
```
```kotlin
// Kotlin
val targetFile = File(context.cacheDir, "bugreport.txt")
val executor = Executors.newSingleThreadExecutor()

Device.requestBugReport(context, targetFile, executor).handle { file, throwable ->
    if (file != null) {
        println("Bug report saved at: ${file.absolutePath}")
    } else {
        println("Error: ${throwable.message}")
    }
}
```

### `requestDumpsys`
- **Summary:** Captures detailed diagnostic dumps of all system services.
- **Behavioral Detail:** 
    - **Content:** Captures the output of the Android `dumpsys` command, providing the internal state of all registered system services.
    - **Resource Lifecycle:** Similar to `requestBugReport`, this is an asynchronous operation that streams data to a file. Existing files are overwritten.
- **Library Level:** `v0.16`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `file` | `File` | `[Mandatory]` | The destination file for the dumpsys output. |
| `executor` | `Executor` | `[Optional]` | Executor for background writing (Defaults to `AsyncTask.THREAD_POOL_EXECUTOR`). |

- **Exceptions:**

| Exception | Condition |
| :--- | :--- |
| `RuntimeException` | Triggered if IPC communication fails or an I/O error occurs while writing to the file. |

- **Example:**
```java
// Java
File targetFile = new File(context.getCacheDir(), "dumpsys.txt");
ExecutorService executor = Executors.newFixedThreadPool(2);

Device.requestDumpsys(context, targetFile, executor)
    .thenAccept(file -> {
        Log.i("library", "System dump completed: " + file.getAbsolutePath());
    })
    .exceptionally(e -> {
        Log.e("library", "Failed to capture dumpsys", e);
        return null;
    });
```
```kotlin
// Kotlin
val targetFile = File(context.cacheDir, "dumpsys.txt")
val executor = Executors.newFixedThreadPool(2)

Device.requestDumpsys(context, targetFile, executor).thenAccept { file ->
    println("Dumpsys saved to: ${file.absolutePath}")
}.exceptionally { throwable ->
    println("Capture failed: ${throwable.message}")
    null
}
```

## 5. DeviceAdmin Class
Manages Device Administrator components and enforces system-wide functional restrictions.

### `isAdminActive`
- **Summary:** Checks if a specific component is currently an active Device Administrator.
- **Behavioral Detail:** Queries the system's active administrator list to verify if the provided `BroadcastReceiver` has been granted administrative privileges.
- **Library Level:** `v0.4`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `admin` | `ComponentName` | `[Mandatory]` | The component of the admin receiver. |

- **Example:**
```java
// Java
boolean active = DeviceAdmin.isAdminActive(context, myAdminComponent);
```

### `setActiveAdmin`
- **Summary:** Silently activates a Device Administrator component.
- **Behavioral Detail:** 
    - **System Impact:** Grants administrative privileges to the specified `BroadcastReceiver` without user interaction.
    - **Usage:** Typically used during initial provisioning or to ensure necessary security policies can be enforced.
- **Library Level:** `v0.4`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `admin` | `ComponentName` | `[Mandatory]` | The component to activate. |
| `refreshing` | `boolean` | `[Optional]` | If `true`, updates an existing active admin (Default: `false`). |

- **Example:**
```java
// Java
DeviceAdmin.setActiveAdmin(context, myAdminComponent, false);
```

### `removeActiveAdmin`
- **Summary:** Revokes administrative privileges from a component.
- **Behavioral Detail:** 
    - **System Impact:** Immediately removes the specified component from the active administrator list. Any security policies previously enforced by this admin are lifted.
- **Library Level:** `v0.4`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `admin` | `ComponentName` | `[Mandatory]` | The component to deactivate. |

- **Example:**
```java
// Java
DeviceAdmin.removeActiveAdmin(context, myAdminComponent);
```

### `setDeviceOwner`
- **Summary:** Designates a component as the administrative owner of the device.
- **Behavioral Detail:** 
    - **Environmental Constraint:** **A Device Owner can only be set once and cannot be removed without a full factory reset.**
    - **System Impact:** Grants absolute administrative control, including deep system policy enforcement and silent app management.
- **Library Level:** `v0.8`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `policyReceiver` | `ComponentName` | `[Mandatory]` | The admin component to be set as owner. |
| `ownerName` | `String` | `[Mandatory]` | Name of the owner organization. |

- **Example:**
```java
// Java

```

### `setProfileOwner`
- **Summary:** Designates a component as the administrative owner of the current user profile.
- **Behavioral Detail:** 
    - **Scope:** Limits administrative control to the current user profile rather than the entire device.
    - **System Impact:** Allows management of profile-specific policies and applications.
- **Library Level:** `v0.8`
- **Parameters:**

| Parameter | Type | Necessity | Description |


## 6. Display Class
Comprehensive control over resolution, orientation, and hardware-specific display attributes.

### `changeOrientation`
- **Summary:** Forces a rotation of the display output.
- **Behavioral Detail:** 
    - **System Impact:** Triggers the system compositor to re-render the screen content at the specified angle (0, 90, 180, 270 degrees).
- **Library Level:** `v0.1`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `displayId` | `int` | `[Mandatory]` | Target display (Primary is typically `0`). |
| `orientation` | `int` | `[Mandatory]` | Rotation value (e.g., `Surface.ROTATION_90`). |

### `setDisplaySize`
- **Summary:** Overrides physical display resolution.
- **Behavioral Detail:** 
    - **Side Effect:** Changing resolution may cause a temporary screen flicker and UI layout disruptions as running applications receive configuration changes and re-draw their interfaces.
- **Library Level:** `v0.10`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `width` | `int` | `[Mandatory]` | Target width in pixels. |
| `height` | `int` | `[Mandatory]` | Target height in pixels. |

- **Example:**
```java
// Java
Display.setDisplaySize(context, 1920, 1080);
```

### `setDisplayDensity`
- **Summary:** Overrides display DPI scaling (density).
- **Behavioral Detail:** 
    - **Side Effect:** Changing density affects the size of all UI elements. Similar to resolution changes, it triggers configuration updates across all active applications.
- **Library Level:** `v0.10`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `density` | `int` | `[Mandatory]` | Target density (DPI). |

- **Example:**
```java
// Java
Display.setDisplayDensity(context, 320);
```

### `DisplayOperation` (Inner Class)
- **Summary:** Unified interface for fine-grained hardware attributes like Brightness and Contrast.
- **Behavioral Detail:** 
    - **Internal Mechanisms:** Provides an abstraction over vendor-specific hardware registers. It automatically enforces valid ranges by querying minimum and maximum values from the underlying platform implementation.
- **Methods:**
    - `getValue()`: Returns the current integer value of the attribute.
    - `setValue(int)`: Sets a new value within the supported range.
    - `getMaximum()` / `getMinimum()`: Returns the hardware-enforced limits.
- **Library Level:** `v0.9`

### `DisplayAttribute` (Enum)
- **Summary:** Defines the specific hardware display attributes that can be adjusted.
- **Behavioral Detail:** Used in conjunction with `DisplayOperation` to identify which specific physical trait of the panel is being queried or modified.
- **Enum Values:**
    - `BRIGHTNESS`: The backlight intensity or overall brightness of the panel.
    - `CONTRAST`: The difference in luminance between the brightest white and the darkest black.
    - `SATURATION`: The intensity or purity of the colors displayed.
    - `HUE`: The general tint or color balance of the display.
    - `SHARPNESS`: The clarity of detail and edge enhancement applied to the image.
- **Library Level:** `v0.9`

## 7. FeatureList Class
Enables discovery and control of supported hardware and software capabilities.

### `getSupportedFeatures`
- **Summary:** Returns a comprehensive list of capabilities supported by the current hardware/software stack.
- **Behavioral Detail:** Aggregates standard Android system features with SOLUM-specific hardware adaptations. **Note that the exact list of supported features may vary depending on the specific device model and hardware configuration.** Each returned `Feature` object is initialized and ready for status queries or modifications.
- **Library Level:** `v0.6`
- **Example:**
```java
List<Feature> features = FeatureList.getSupportedFeatures(context);
```

### `Feature` (Inner Class)
- **Summary:** Represents a specific system or hardware feature that can be toggled or queried.
- **Behavioral Detail:** 
    - **Usage:** Instances must be obtained via `FeatureList.getSupportedFeatures()`. Calling `get()` or `set()` on an uninitialized instance will throw an `IllegalStateException`.
    - **Enum Values:**
        - `BLOCK_REMOTE_CONTROL`: Disables input from the infrared (IR) remote control.
        - `BLOCK_REMOTE_CONTROL_EXCEPTS_POWER`: Disables IR remote control input, except for the power button.
        - `BLOCK_REMOTE_CONTROL_WITHOUT_PASSWORD_POPUP`: Silently blocks IR remote input without displaying a password prompt UI.
        - `BLOCK_TOUCH_EVENT`: Disables input from the touch screen.
        - `BLOCK_USB_POWER`: Cuts off power to the external USB ports.
        - `BLOCK_HDMI_IN`: Disables the display of external HDMI input sources.
        - `BLOCK_RS232_CONTROL`: Disables system control commands via the RS232 serial port.
- **Methods:**
    - `get()`: Returns the current status value of the feature (typically `0` for disabled, `1` for enabled).
    - `set(int value)`: Updates the feature status. Changes are applied immediately at the system level.
- **Library Level:** `v0.6`
- **Example:**
```java
// Java
List<Feature> features = FeatureList.getSupportedFeatures(context);
for (Feature feature : features) {
    if (feature == Feature.BLOCK_USB_POWER) {
        // Query current status
        int status = feature.get();
        // Toggle feature (1 to enable block, 0 to disable)
        feature.set(status == 1 ? 0 : 1);
    }
}
```

## 8. Input Class
Simulates raw user interaction events at the system level.

### `injectInputEvent`
- **Summary:** Injects touch, mouse, or keyboard events into the system input stream.
- **Behavioral Detail:** 
    - **Synchronization Modes:**
        - `ASYNC`: Non-blocking; fire-and-forget.
        - `WAIT_FOR_RESULT`: Waits for the dispatcher to confirm the event is permitted based on current focus.
        - `WAIT_FOR_FINISH`: Blocks until the active application has fully handled the event.
    - **Side Effect:** Events are treated as genuine physical input, affecting the behavior of any focused application.
- **Library Level:** `v0.5`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `event` | `InputEvent` | `[Mandatory]` | The `KeyEvent` or `MotionEvent` to inject. |
| `mode` | `int` | `[Optional]` | Injection synchronization mode (Default: `ASYNC`). |

- **Example:**
```java
// Java: Injecting a Home key press
KeyEvent downEvent = new KeyEvent(KeyEvent.ACTION_DOWN, KeyEvent.KEYCODE_HOME);
Input.injectInputEvent(context, downEvent, 1); // WAIT_FOR_RESULT
```

## 9. KeyStore Class
Performs operations related to the device's internal security keys.

### `installTrustedAnchor`
- **Summary:** Installs a Root CA certificate into the system's trusted credentials store.
- **Behavioral Detail:** 
    - **Side Effect:** Enables secure SSL/TLS communication with private or internal servers without "Untrusted Root" warnings. This affects all applications on the device that rely on the system trust store.
    - **Persistence:** Certificates installed via this API persist across reboots but are removed upon a factory reset.
- **Library Level:** `v0.10`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `caUri` | `Uri` | `[Mandatory]` | The URI of the certificate file (X.509 format). |

- **Exceptions:**

| Exception | Condition |
| :--- | :--- |
| `IOException` | If the certificate file cannot be read or exceeds the size limit (10MB). |

- **Example:**
```java
// Java
Uri certUri = Uri.fromFile(new File("/sdcard/my_root_ca.crt"));
KeyStore.installTrustedAnchor(context, certUri);
```

## 10. Locale Class
Management of system-wide language and network time synchronization.

### `setLocale`

## 11. Package Class
Advanced application lifecycle management, including silent installation.

### `install`
- **Summary:** Performs a silent application installation or update using a file path or URI.
- **Behavioral Detail:**
    - **System Impact:** Silently installs or updates the application, completely bypassing the standard Android package installer UI and user confirmation dialogs.
    - **Input Support:** Supports both absolute **file paths** and **URIs** (e.g., from a `FileProvider`). When a `Uri` is used, the library automatically grants the necessary read permissions to the system service.
    - **Lifecycle Transitions:** `Started` -> `Writing` -> `Committing` -> `Finished`.
- **Library Level:** `v0.1`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `filePath` / `apkUri` | `String` / `Uri` | `[Mandatory]` | The location of the APK file. |
| `listener` | `InstallListener` | `[Optional]` | Callback interface to monitor progress and status. |

- **Example:**
```java
// Java: Installing via File Path
Package.install(context, "/sdcard/my_app.apk", myListener);

// Java: Installing via Uri (Recommended for Android 7.0+)
Uri apkUri = FileProvider.getUriForFile(context, "com.example.provider", apkFile);
Package.install(context, apkUri, myListener);
```

### `grantPermission`
- **Summary:** Silently grants runtime permissions to a specific package.
- **Behavioral Detail:** 
    - **System Impact:** Modifies the system's permission database directly, eliminating the need for the user to see or approve runtime permission requests.
- **Library Level:** `v0.1`
- **Parameters:**

| `packageName` | `String` | `[Mandatory]` | Target application package name. |
| `permissions` | `String[]` | `[Mandatory]` | Array of permission strings (e.g., `Manifest.permission.CAMERA`). |

- **Example:**
```java
// Java
String[] perms = { "android.permission.CAMERA", "android.permission.ACCESS_FINE_LOCATION" };
```

## 14. ScreenCapture Class
High-fidelity display content acquisition using system and hardware-level methods.

### `isHardwareScreenCaptureSupported`
- **Summary:** Verifies the device's capability for direct framebuffer screen capture.
- **Behavioral Detail:** Determines if the current hardware and platform layer allow reading raw pixel data directly from the framebuffer. This is required for using `requestHardwareScreenCapture`.
- **Library Level:** `v0.17`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |

- **Example:**
```java
// Java
if (ScreenCapture.isHardwareScreenCaptureSupported(context)) {
    Log.i("library", "Hardware screen capture is available.");
}
```

### `requestScreenCapture`
- **Summary:** Captures the current display content using Android system services.
- **Behavioral Detail:** 
    - **Standard Capture:** Utilizes the Android Compositor (SurfaceFlinger) to capture all visible UI layers, including system overlays and notification bars.
- **Library Level:** `v0.17`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `displayId` | `int` | `[Optional]` | Target display ID (Default: `0`). |

- **Example:**
```java
// Java
Bitmap screenshot = ScreenCapture.requestScreenCapture(context, 0);
```

### `requestHardwareScreenCapture`
- **Summary:** Captures the raw framebuffer output for 100% fidelity.
- **Behavioral Detail:** 
    - **Unique Advantage:** Bypasses high-level rendering services to capture the exact final pixel output. This is the **only method** to capture content from **HDMI-In** sources and hardware-level **OSD** elements that are invisible to standard screen capture.
    - **Async Operation:** This is an asynchronous operation. It returns a `Future` object that will eventually contain the `File` where the screenshot was saved.
- **Library Level:** `v0.17`
- **Parameters:**

| Parameter | Type | Necessity | Description |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | `[Mandatory]` | Caller context. |
| `fileName` | `String` | `[Mandatory]` | Full path where the raw capture should be saved. |

- **Returns:** `Future<File>` - A handle to the asynchronous capture task.

- **Example:**
```java
// Java: Handling the asynchronous hardware capture
if (ScreenCapture.isHardwareScreenCaptureSupported(context)) {
    Future<File> task = ScreenCapture.requestHardwareScreenCapture(context, "/sdcard/raw_capture.png");
    
    // You can handle the result on a background thread
    new Thread(() -> {
        try {
            File result = task.get(); // Blocks until the capture is complete
            Log.i("library", "Hardware capture saved to: " + result.getAbsolutePath());
        } catch (Exception e) {
            Log.e("library", "Hardware capture failed", e);
        }
    }).start();
}
```
